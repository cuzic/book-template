# 2025-12-07: エージェントチーム構成の初期設定

## 背景

書籍執筆を効率化するため、7つの役割からなるエージェントチームを構成しました。

## チーム構成

| 役割 | 責務 | モデル |
| ---- | ---- | ------ |
| Author | 全体統括、意思決定 | opus |
| Researcher | Web検索、知見収集 | sonnet |
| Writer | 章の執筆 | sonnet |
| Reviewer | 批判的レビュー | sonnet |
| Editor | 文体統一、推敲 | sonnet |
| Publisher | ビルド、検証 | sonnet |
| Architect | スキル・プロセス改善 | opus |

## 設計判断

### ファイル所有の分離

各エージェントが担当するファイルを明確に分離しました。

- 同じファイルを複数エージェントが編集すると競合が発生するため
- WriterからEditorの順序で章ファイルを編集する運用

### モデル選択

- Author/Architect: 意思決定と創造性が必要なためopus
- その他：タスク実行中心のためsonnet（コスト効率）

### スキル設計

5つのスキルを用意しました。

- `book-outline`: 目次生成・改善
- `book-research`: Web検索・知見保存
- `book-review`: 批判的レビュー
- `book-apply`: レビュー結果適用
- `book-write`: 章の執筆
