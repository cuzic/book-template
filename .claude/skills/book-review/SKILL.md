# Devil's Advocate レビュースキル

## 使い方

- `/book-review` - 目次全体をレビュー
- `/book-review --iterations 3` - レビュー→改善を3回繰り返す
- `/book-review --chapter 01` - 特定の章をレビュー

## オプション

$ARGUMENTS

## レビュー観点

批判的な視点で以下を評価する:

1. **構成の論理性**: 章の順序、依存関係は適切か
2. **網羅性**: 重要なトピックが漏れていないか
3. **冗長性**: 不要な重複はないか
4. **対象読者との整合性**: 難易度、前提知識は適切か
5. **独自性**: 類書と比較した差別化ポイントはあるか
6. **実用性**: 読者が実際に活用できる内容か
7. **図解の必要性**: 視覚的説明が有効な箇所はあるか

## 処理手順

1. 現在の `src/chapters/` の構造を読み込む
2. `book.json` のメタデータを確認する
3. 批判的な視点で問題点を洗い出す
4. 改善提案をリスト化する
5. `knowledges/reviews/YYYY-MM-DD-review.md` に結果を保存する

## 出力フォーマット

```markdown
# 目次レビュー結果

レビュー日: YYYY-MM-DD

## 評価サマリー

| 観点 | 評価 | コメント |
|------|------|----------|
| 構成の論理性 | B | 〜 |
| 網羅性 | A | 〜 |

## 問題点

### 問題1: 〜が不足している

- 詳細: ...
- 影響: ...
- 該当箇所: 第X章

## 改善提案

1. 【優先度:高】〜を追加する
2. 【優先度:中】〜の順序を入れ替える
3. 【優先度:低】〜を削除または統合する

## 図解提案

図解を入れた方が分かりやすい箇所を特定する。

### 図解1: [タイトル]
- **該当箇所**: 第X章、セクションY
- **種類**: プロセス図 / 比較表 / 構造図 / フロー図 / 概念図
- **理由**: なぜ図解が有効か
- **推奨アスペクト比**: 3:4 / 1:1 / 4:3

## 次のアクション

- [ ] 改善提案1を適用
- [ ] 改善提案2を適用
- [ ] 図解提案をIllustratorに依頼
```

## 注意事項

- 建設的な批判を心がける
- 具体的な改善案を提示する
- 優先度を明確にする
